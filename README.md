# 1. Titanic Survival Analysis
Overview

This project analyzes the famous Titanic dataset to uncover survival patterns based on demographic, economic, and social factors. The analysis explores passenger behavior, ticket classes, fare distribution, family/group travel patterns, and survival outcomes.

Objectives
Understand factors influencing survival
Analyze passenger demographics
Explore fare and embarkation trends
Identify family and group travel behavior
Key Insights
Gender and passenger class were the strongest predictors of survival.
Women and children, especially in 1st class, had the highest survival rates.
Men traveling in 3rd class had the lowest chances of survival.
Fare and embarkation port acted as indirect indicators of wealth and social status.
Social inequality significantly influenced survival outcomes during the disaster.
Key SQL Queries
Average age of passengers by passenger class
Survival rate by gender and passenger class
Total fare collected from each embarkation port
Passengers traveling alone
Passengers sharing the same ticket number
# 2. Library Management System Analysis
Overview

This project examines a library database to understand book demand, monitor missing books, and optimize storage management. The analysis focuses on improving operational efficiency and resource allocation.

Objectives
Track book demand and circulation
Identify missing or high-risk inventory
Optimize cabinet and rack utilization
Analyze user activity trends
Key Insights
Certain categories showed a higher proportion of missing books, indicating possible security or tracking issues.
Some books were checked out multiple times in a single day, highlighting strong user demand.
The top 3 authors accounted for a significant percentage of total checkouts.
Cabinet and rack analysis identified:
Weak signal strength areas
Frequently missing inventory zones
Completely unused storage spaces
Peak checkout times revealed user activity trends useful for staffing decisions.
Key SQL Queries
Books checked out multiple times in one day
Most popular category based on checkouts
Cabinets with the highest proportion of checked-out books
Cabinets with the weakest average signal strength
Unused cabinet/rack/row combinations
# 3. Zomato Data Analytics Project
Overview

This project analyzes food ordering behavior, restaurant performance, and customer trends using SQL. The objective is to uncover insights that can help improve customer retention, optimize operations, and increase revenue.

Objectives
Analyze customer ordering behavior
Identify top-performing restaurants
Understand revenue-driving food categories
Track repeat customer activity
Evaluate delivery performance
Key Insights
Premium and highly rated restaurants consistently generated higher sales.
Restaurants with better ratings received more orders and produced greater revenue.
Fast-food items such as Pizza, Biryani, and Burgers dominated customer orders.
Repeat customers had higher Average Order Value (AOV) and contributed significantly to total revenue.
Most orders occurred between 7 PM and 10 PM, indicating peak dinner-time demand.
A small segment of loyal customers accounted for a major share of revenue, making them ideal candidates for loyalty programs.
Key SQL Queries
Restaurant with the highest total sales value
Customer segmentation by spending behavior
Repeat customers ordering multiple times in a month
Average delivery time per delivery partner
